Supabase 初期設定

複数デバイスでデータを共有するための設定です

手順（5分で完了）
  1. supabase.com でプロジェクトを作成
  2. Settings → API → Project URL と anon public キーをコピー
  3. SQL Editor → New query → 以下を貼り付けて Run:create table public.chips ( id text primary key, user_id uuid references auth.users not null, number text not null default '', name text not null default '', cost numeric not null default 0, price numeric not null default 0, status text not null default 'unlisted', size text not null default '', description text not null default '', images jsonb not null default '[]', created_at text not null default '', updated_at text not null default '' ); create table public.sales ( id text primary key, user_id uuid references auth.users not null, date text not null default '', chip_id text, chip_name text not null default '', chip_number text not null default '', price numeric not null default 0, cost numeric not null default 0, note text not null default '', created_at text not null default '' ); create table public.expenses ( id text primary key, user_id uuid references auth.users not null, date text not null default '', category text not null default '', amount numeric not null default 0, description text not null default '', created_at text not null default '' ); alter table public.chips enable row level security; alter table public.sales enable row level security; alter table public.expenses enable row level security; create policy "own chips" on public.chips for all using (auth.uid() = user_id) with check (auth.uid() = user_id); create policy "own sales" on public.sales for all using (auth.uid() = user_id) with check (auth.uid() = user_id); create policy "own expenses" on public.expenses for all using (auth.uid() = user_id) with check (auth.uid() = user_id);
  4. Authentication → Providers → Google を有効化
  5. Authentication → URL Configuration → Redirect URLs に GitHub Pages URL を追加
    （例: https://あなたのID.github.io/nail-manager）
